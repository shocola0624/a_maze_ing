## Description

本プロジェクトは、迷路の自動生成からターミナル上でのレンダリングし、16進数でのファイル出力までを行うPythonプログラムを作成し、モジュール化したものである。

### 開発目的

仕様を満たすだけでなく、保守しやすい設計やチーム開発の基本を実践することを意識した。

- **データ構造と処理の工夫**: 迷路データは二次元配列で扱い、出力時はビットマスクで圧縮。描画やファイル保存はメモリ上でまとめてから一気に出力し、処理負荷を減らした。
- **ファイル分割とエラー対策**: 「計算」「描画」「入出力」でファイルを分割。想定外の入力でもプログラムが落ちないよう、基本的な例外処理を入れた。
- **Issueの使用**: GitHubのIssueでタスクを細分化。Pull Requestには「テスト手順と結果」を明記するルールにし、お互いの作業状況を把握しやすくした。

## Bonus Features

仕様書の必須要件に加え、本プロジェクトでは以下のボーナス機能を実装している。

- **アニメーション:** 探索アルゴリズムが壁を伸ばしていく過程をターミナル上でリアルタイムにアニメーション表示する。
- **描画速度の変更**`config.txt` に `WAIT_SEC` を設定し、数値を変更することでユーザーが描画速度を自由に調整できる。
- **実行中のファイル内容の変更:**迷路生成スクリプト実行中にconig.txtの中身を書き換えても、リアルタイムに処理に反映される。
- **カスタムカラー:** 4種類のカラーテーマの循環変更や、床・壁・42・入口・出口・経路ごとの個別カラー割り当てが可能である。
- **任意のシード値指定:** 「シードによる再現性」という必須要件を拡張し、`config.txt` からユーザーが手動で任意の `SEED` 値を指定・変更できる仕様にした。これにより、気に入った特定の迷路パターンを簡単に保存・共有・再現できる。

## Instructions

### 環境

- Python 3.10+

### Setup & Execution

本プロジェクトは、レビュアーのローカル環境を汚さないよう、自動的に仮想環境（venv）を構築して実行される安全な設計になっている。

1. インストール（仮想環境の構築）

```bash
make install
```

自動的に venv フォルダが作成され、その隔離環境内に必要なパッケージと開発ツール（flake8, mypy, build）がインストールされる。

2. プログラムの実行

```bash
make run
```

設定ファイル（config.txt）を読み込み、迷路の生成およびターミナルへの描画を行う。（仮想環境内のPythonが自動で呼び出される）

3. パッケージのビルド（再利用性要件）

```bash
make build
```

dist/ ディレクトリ下に、本プログラムをライブラリとして配布するための .tar.gz および .whl ファイルを生成する。

4. デバッグの実行

```bash
make debug
```

5. コードチェック

```bash
make lint
make lint-strict
```

隔離環境にインストールされた flake8 と mypy を使用し、本プロジェクトの対象コードのみを確認する。

6. プロジェクトの初期化

```bash
make clean
```

生成されたキャッシュや、venv、ビルド成果物（dist/）をすべて削除し、綺麗な状態に戻す。

## Configuration file format

設定ファイル（`config.txt`）は以下のキーと値で構成され、迷路構造と生成の挙動を制御する。

- `WIDTH`: 迷路の幅（セル数）
- `HEIGHT`: 迷路の高さ（セル数）
- `ENTRY`: 入口の座標 (x, y)
- `EXIT`: 出口の座標 (x, y)
- `OUTPUT_FILE`: 生成された迷路（16進数形式）を出力するファイル名
- `PERFECT`: `True` の場合、ループを持たない迷路を生成する
- `SEED`: （任意）迷路生成の乱数シード値。指定しない場合はランダム生成される。
- `WAIT_SEC`: （任意）ターミナル描画時の1ステップの待機時間。0より大きい値の時、アニメーションが有効になる。

## Project Structure & File Roles

プロジェクトは役割ごとにディレクトリが分割されている。  
各モジュールと主要メソッドの役割は以下の通り。

### 1. Core Entry Point

- `a_maze_ing.py`
- **役割**: プロジェクトのメインスクリプト。
- **主要処理**: `config_utils` で設定を読み込み、`maze_utils` の各機能を順番に呼び出して、ファイル出力までの一連の処理を制御する。

### 2. config_utils/

設定ファイルの読み込みと、エラーハンドリング（バリデーション）を担うモジュールである。

- `keys.py`
  - **役割**: 辞書のキー名（WIDTH, HEIGHTなど）を Enumクラスを用いて一元管理する。
  - **意図**: 文字列のハードコードによるエラーを防ぐ（タイポとか）。

- `get_config_path.py`
  - **役割**: コマンドライン引数の安全な取得と事前バリデーション。
  - `get_config_path()`: sys.argv を用いて、プログラム実行時に渡された引数の数を確認する。引数が1つ（要素数が2つ）でない場合は、標準エラー出力（stderr）にエラーメッセージを出力し、プログラムを即座に強制終了（exit）する。正しい場合はファイルパスの文字列を返す。

- `read_config.py`
  - **役割**: 設定ファイルの安全な読み込みと、辞書型データへのパース処理。
  - `read_config()`: 以下のアプローチを用いて、テキストファイルを解析し、config_data 辞書を生成する。
    1. ノイズ除去: 空行や # で始まるコメント行をスキップする。
    2. フォーマット検証: 各行が KEY=VALUE 形式であるかを確認する。フォーマット違反（= が無い、または複数ある等）があれば、標準エラー出力。
    3. フェイルオーバー: FileNotFoundError や ValueError を、try-exceptでクラッシュさせず、安全に `None` を返し、後ろの処理にエラー渡す。

- `validate_config.py`
  - **役割**: ユーザー入力に対するバリデーションと型の正規化。
  - `validate_config()`: 後ろの迷路生成処理が安全に実行できるよう、以下の5つの処理でデータをバリデーションする。
    1. 構造チェック: 必須キーの欠如や、未知のキーの混入がないか検証し、生の文字列キーを安全な Enum型に変換する。
    2. サイズチェック: WIDTH と HEIGHT が正の整数であることを保証し、文字や負の数によるクラッシュを防ぐ。
    3. 座標の境界チェック: ENTRY と EXIT の文字列をタプルにパースし、「迷路の枠外にはみ出していないか」「スタートとゴールが重複していないか」を検証する。
    4. 干渉チェック: 中央に配置される「42」の領域に、スタートやゴールが設定されていないかを事前に検知して弾く。
    5. 値の正規化: PERFECT(True/False) や WAIT_SEC (float) 等の入力を、プログラムが扱いやすい型へキャストする。

### 3. maze_utils/

迷路の生成・探索・フォーマット変換を担う、本プロジェクトのメインロジックである。  
ファイルの読み書きや画面への表示といった処理（I/O）とは責任分離し、「設定データを受け取って、迷路を作って返すだけ」の独立したモジュールとして設計した。これにより、CLIツールとしてだけでなく、将来的にWebアプリや別のシステムに組み込む際にも、コードを書き直すことなく再利用できる設計になっている。

- **`generate_maze.py`**
- **役割**: configファイルによる条件を満たし、かつクリア可能な迷路を生成すること。
  - **処理フロー**: 処理は大きく分けて、以下の二つ。
  1. データ構造の作成：辞書より、数字の二次元配列をいじる。
  2. データの可視化：完成した数字の配列を、色付きのブロックに変換
  - **`generate_expanded_maze()`**:
  - **役割**: 迷路生成の全体を指揮するメイン関数。
    - **処理フロー**: 以下の手順で、段階的にデータを加工・完成させる。
    1. **二次元配列の確保**: 辞書のデータを受け取り、ベースとなる拡張グリッド（0埋めの2次元配列）を作成する。
    2. **初期設定**: `initialize_maze` を呼び出し、外周枠やスタート・ゴール、固定オブジェクトを二次元配列に書き込む。
    3. **壁の生成**: `set_walls` を呼び出し、生成アルゴリズムに従って二次元配列に壁を書き込む。
    4. **経路探索**: `find_shortest_path` を呼び出し、正解ルートを計算して配列に書き込む。
    5. **出力**: 完成した配列データを返し、後の描画処理（`print_maze`）やファイル出力へ渡す。

- **`initialize_maze()`**:
  - **役割**: 二次元配列に対し、迷路生成の土台となる外周やスタート・ゴール、固定オブジェクトを配置する関数。
    - **処理フロー**: 以下の手順で二次元配列を初期化する。
    1. **スタート・ゴールの配置**: 設定ファイルで指定された座標を、拡張された二次元配列上の通路になる奇数座標（x\*2+1）に変換し、入口出口（4,5）を書き込む。
    2. **外周の配置**:
       キャンバスの上下左右の外周をすべて「壁（1）」で囲みます。これは、後続の壁生成アルゴリズムが二次元配列の外にはみ出してクラッシュしないためのガードとして機能する。
    3. **固定オブジェクト「42」の配置と保護**:
       キャンバスが十分な広さを持つ際、中央に「42」のピクセルアート（2）を書き込む。この時、文字の周囲8マスを（3）で書き込む。これにより、後からランダムに成長する壁による破壊を防ぐ。

  - **`set_walls()`**:
    - **役割**: 迷路全体に壁を張り巡らせるため、42オブジェクト周りに「起点」を決定し、壁の成長を制御する関数。
      - **処理フロー**: 以下の手順で壁の起点を抽出し、生成を行う。
      1. **42オブジェクトの事前接続（Perfect設定時のみ）**: ランダムな壁生成を始める前の下準備。ループのない完璧な迷路を作る場合、巨大な「42」オブジェクトが周りを道に囲まれて孤島になるのを防ぐ必要がある。そのため、まずは「42」の周囲を最初の起点に選び、外枠に向かって壁の橋を伸ばし、迷路全体と陸続きにしておく。
      2. **全起点候補のリストアップ**:
         キャンバス（二次元配列）全体を走査し、これから壁の柱を立てることができるすべての「偶数・偶数座標」をリストアップして配列に格納する。
      3. **ランダムな壁の成長ループ**:
         2で作ったリストから、ランダムに座標を1つ取り出す。その場所がまだ「床（0）」であれば新しく「壁（1）」を置き、そこを新たな起点として `grow_wall_from`（壁を伸ばす関数）を呼び出す。すべての柱候補をチェックし終える（リストが空になる）までこのループを繰り返し、隙間なく壁を伸ばす。

  - **`grow_wall_from()`**:
    - **役割**: 指定した座標から、迷路の壁を伸長させる関数。
      - **処理フロー**: 以下の手順を反復し、壁を伸ばす。
      1. **拡張可能方向の抽出**: 現在の座標から上下左右の方向を評価し、壁の伸長が可能な方向をリストアップする。通常モードでは閉路の発生を防ぐため「2マス先の座標が未確保（0）」であることを拡張条件とする。ただし、既存の壁への接続を目的とする to_wall モードでは「隣接マスが未確保」であれば拡張候補に含める。
      2. **経路履歴の保存**: バックトラック処理用に、現在の座標（coord）を履歴（taken_coords）で保持する。
      3. **壁の伸長と再帰呼び出し**: 抽出した候補からランダムに1つの方向を選択し、通路部分を跨いだ2マス分を壁（1）として書き込む。到達した2マス先の座標が未確保領域であった場合、その座標を新たな起点として自身を再帰的に呼び出し、壁の伸長を継続する。もし到達先が既存の壁であった場合、処理を終了する。
      4. **行き止まり時のバックトラック**: 進行可能な方向が存在しない（行き止まり）状態であり、かつ to_wall モードで実行されている場合、保持した履歴（taken_coords）から過去の座標をランダムに抽出し、既存の壁に到達するまで別ルートからの伸長を行う。

  - **`find_shortest_path()`**:
    - **役割**: 完成した迷路データに対し、幅優先探索を実行し、入口から出口までの最短経路を特定・描画（データ上書き）する関数である。
      - **処理フロー**: 以下ので経路を計算し、配列を更新する。
      1. **探索キューと履歴の初期化**: 探索の起点となる入口座標を（coords）に登録する。同時に、各座標の「1つ手前の座標（親ノード）」を記録するための辞書（prev）を初期化し、経路の重複探索を防ぐ。
      2. **幅優先探索（BFS）の実行**: キュー（FIFO）を用いて、入口から近い座標順に探索する。現在地から到達可能な隣接マス（床 0 または 出口 5）を評価し、未探索であればキューの末尾に追加する。同時に、 prev辞書に対して「進んだ先のマスをキー、元いたマスを値」として記録し、後で経路を逆算するためのログを残す。これを出口に到達するまで繰り返す。
      3. **経路の逆算（バックトレース）とマーキング**: 探索が出口に到達した時点で、`prev` 辞書のポインタを利用し、出口から入口へ向かってノードを逆順に遡る。この最短経路上の座標（床 0）に対して、経路マーカー（6）の数値を上書きし、正解ルートを二次元配列内に書き込む。

  - **`print_maze()` / `clear_screen()`**:
    - **役割**: 2次元配列のデータを視覚的な迷路としてターミナルに描画する関数。迷路に色をつけ、処理負荷を減らしてアニメーションを実装した。
      - **処理フロー**:
      1. **数値から色指定文字へ変換**: 配列内の数値（0〜6）を、対応する背景色のエスケープ文字列に置き換える。
      2. **バッファリング出力**: print関数を用いた画面への出力処理は、コンテキストスイッチが発生するため、処理が重い。したがって、CPUのメモリ上で先に1画面分の巨大な文字列を全て結合しておき、最後に出力処理を走らせ、計算コストを削減した。
      3. **パラパラ漫画の原理（画面リセット機能）**: アニメーション描画の際は、出力の直前に `clear_screen()` を呼び出す。ここで使われる "\x1b[2J\x1b[H" という文字列は、以下の2つの命令を組み合わせたものである。
      - "\x1b[2J": ターミナルの画面を全て消去する。
      - "\x1b[H": 文字を書き始めるカーソル位置を、画面の左上に戻す。  
        これを「待機（sleep）」と組み合わせて「消す → 左上から新しい迷路を書く → 待つ」を繰り返し、アニメーションを実現した。

- **`convert_maze.py`**
- **役割**: 迷路生成用の二次元配列を解析し、最終的な出力要件に合わせたデータ構造（元のグリッドサイズとビットマスク表現）へと圧縮するモジュールである。
  - **処理フロー**: 以下の手順で巨大な配列を圧縮する。
  1. **ビットマスクの定義**: 各方向の壁を表現するため、北(1), 東(2), 南(4), 西(8) の2の乗数として定義する（ビットマスク）。これにより、4方向の壁の組み合わせを(0〜15)で一意に表現できる。
  2. **拡張グリッドの走査**: `convert_maze()` にて、二次元配列のすべての「通路/部屋（奇数座標）」を走査する。
  3. **壁情報のエンコード（`func`）**: 各部屋の座標から隣接する上下左右のマスを確認する。対象マスが壁(1)、固定オブジェクト(2)、保護領域(3)のいずれかである場合、その方向に壁が存在すると判定し、対応する（1, 2, 4, 8）を加算する。この計算結果を、元の指定サイズに縮小した新しい2次元配列（`converted_maze`）の対応する座標に格納し、最終的な出力用データとして返す。

- **`get_shortest_path.py`**
- **役割**: `generate_maze.py` によって二次元配列に書き込まれた最短経路（6）をたどり、方向指定文字列（N, E, S, W）に変換・出力する翻訳モジュールである。
  - **処理フロー**: 以下の手順で経路を辿り、文字列を生成する。
  1. **方向と移動量のマッピング**: 4方向の文字列キー（N,E,S,W）と、それに対応するXY座標の移動量を紐付けた辞書（adj）を定義し、探索の基準とする。
  2. **経路のトレースと逆走防止**: 入口座標（ENTRY）を起点とし、周囲4方向のマスから最短経路（6）を探索する。この時、直前に記録した方向の逆方向（oposite辞書による判定）を探索候補から除外する（例: 直前がSならNは調べない）。これにより、Uターンや無限ループを防止する。
  3. **文字列の結合と座標更新**: 隣接座標に（6）を発見した場合、対応する方向の文字列を取得して経路文字列（path）の末尾に結合する。その後、現在地を2マス先（次の柱座標）へ更新する。この処理を出口座標（`EXIT`）に到達するまでループし、最終的な最短経路文字列を返す。

- **`output_maze.py`**
- **役割**: 迷路データ（ビットマスク値）、スタート・ゴール座標、最短経路文字列を、指定されたフォーマットに加工し、外部ファイルへ出力するモジュールである。
  - **処理フロー**: 以下の手順で出力用文字列へ加工し、ファイルへの書き込みを行う。
  1. **16進数へのエンコード**: `convert_maze.py` で圧縮された(0〜15)の整数値を、`int_to_hex` 関数を用いて (0〜F)の16進数文字に変換する。
  2. **データ整形とバッファリング（I/O負荷の削減）**: ディスクのI/O回数を減らすため、メモリ上で全出力データを1つの文字列変数に結合する。また、タプル型の座標データにスライス操作を行い、括弧を含まない形式"0,5"に加工して連結する。最後に、探索済みの経路文字列（例: "NNEESW"）を改行区切りで追加する。
  3. **ファイルへの一括書き出し**: 迷路データの構築が完了した後、指定されたファイルパス（OUTPUT_FILE）を書き込みモードで展開し、メモリ上の文字列を一括で出力する。処理完了後は解放する。

- **`ask_next_process.py`**
  - **役割**: 迷路の生成および初回描画の完了後に、プログラムを終了させずターミナル上でユーザーからのコマンド入力を待機し、対話的に画面表示を操作するためのUIモジュールである。
  - **処理フローと技術的工夫**: 以下の機能を提供し、入力に応じた状態の更新と画面の再描画を実行する。
    1. **再帰による入力待機**:
       whileループによる繰り返しではなく、関数の最後で自分自身（`ask_next_process`）を再度呼び出す再帰処理によって入力待ちを継続している。この際、ユーザーが変更した設定（経路表示のON/OFFや、配色番号など）を引数として次の関数へ渡すことで、直前の状態を安全に保持したまま新しい画面を描画し続ける仕組みとなっている。
    2. **バリデーションチェック**:
       設定の選択時、入力された文字が数字に変換できない（ValueError）を、プログラムの終了（Quit）として利用している。また、配色設定では存在しない色名によるエラー（KeyError）では、正しい値が入力されるまで質問を繰り返すバリデーションを実装。
    3. **剰余演算**:
       4種類の配色テーマを順番に切り替える際、現在のインデックス番号をテーマの総数で割った余り（scheme_index % len）を利用している。これにより、何度切り替えボタンを押しても配列の範囲外エラー（IndexError）が発生せず、安全に循環する設計である。

## Maze generation algorithm

本プロジェクトでは、要件である「ループのない完璧な迷路」の生成と、「最短経路の探索」を実装するため、以下のアルゴリズムを選定した。

### 1. 迷路生成：深さ優先探索（DFS / バックトラック法）

- **選定理由**: 実装がシンプルであり、迷路構造を作りやすいアルゴリズムだからである。
- **特徴**: ランダムな方向へ進めるだけ進み、行き止まりにぶつかると「まだ進める別の道」まで履歴を遡る性質がある。これにより、空間を余すことなく使い切り、仕様書の要件である「ループを持たない完璧な迷路」を生成することができる。

### 2. 経路探索：幅優先探索（BFS）

- **選定理由**: 今回のような各マスの移動コストが同じ迷路において、確実に最短経路を見つけ出すことができるアルゴリズムだからである。
- **特徴**: 入口から近いマスから順番に、同心円状に探索を広げていく。最初に出口に到達したルートが必然的に最短経路となるため、直感的に実装できる。

## Code Reusability（モジュールの再利用性）

本プロジェクトは、`a_maze_ing.py`と、迷路生成の処理（`maze_utils` および `config_utils`）を分離（疎結合）して設計した。これにより、迷路生成アルゴリズムとして安全に再利用することが可能である。

外部プロジェクトで利用する場合は、以下の手順。

### Step 1: パッケージのビルド
プロジェクトのルートディレクトリで以下のコマンドを実行し、配布用のパッケージを作成する。
```bash
make buildkv
```
* 目的:
`pyproject.toml` の設定に基づき、再利用可能なモジュール群（`maze_utils`, `config_utils`）のみを `.whl`（Wheel）ファイルとして独立したパッケージに梱包する。

### Step 2: 外部環境へのインストール
生成された `.whl` ファイルを、利用したい別の仮想環境にインストールする。

```bash
python3 -m venv test_venv
source venv/bin/activate
pip install /path/to/a_maze_ing/dist/mazegen-2026.3.28-py3-none-any.whl
```
* 目的:
このモジュールをシステムの標準ライブラリと同じように、どこからでも `import` できる状態にする。

### Step 3: 外部からの呼び出し
インストール完了後、外部のPythonコードから以下のようにインポートして使用する。テキストファイル（`config.txt`）の読み込みに依存せず、辞書型データを使って直接迷路を生成できる。

```python
from maze_utils import MazeGenerator, get_shortest_path

def main():
    # 1. パラメータの指定
    config_data = {
        "WIDTH": 10,
        "HEIGHT": 10,
        "ENTRY": (0, 0),
        "EXIT": (9, 9),
        "PERFECT": True,
        "WAIT_SEC": 0.0,
        "SEED": 42
    }

    # 2. 迷路の生成と構造データへのアクセス
    gen = MazeGenerator()
    expanded_maze = gen.generate_expanded_maze(config_data)

    # 3. 最短経路の取得
    shortest_path = get_shortest_path(expanded_maze, config_data)
    print(f"Shortest Path: {shortest_path}")

    # ターミナルに描画して確認
    gen.print_maze(expanded_maze, config_data)

if __name__ == "__main__":
    main()
```
